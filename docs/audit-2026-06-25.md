# SAMAR Audit — 2026-06-25

This document captures findings from a full audit of the SAMAR codebase
conducted on 2026-06-25, immediately after the reorganization commits
`6431347` and `5c5da42`. All 20 findings were fixed in a single follow-up
commit (`6d15b00`). A second round on the same day found and fixed 5
additional critical bugs that round 1 missed; details in
`docs/audit-round-2.md`. Where the original FIGARO paper provided a
canonical design choice, SAMAR follows FIGARO; where FIGARO doesn't apply
(SAMAR-specific 24-EDO extension), the rationale is documented inline.

## Critical bugs (fixed)

### #1 Two parallel `MusicXMLParser` implementations

**WHAT**: `samar/parser.py` and `samar/core.py` both defined `class MusicXMLParser`.
The `parser.py` version hardcoded 4/4 bar length; the `core.py` version
honored per-bar time signatures.

**WHERE**: Production code (`samar/dataset.py → samar/input_representation.py
→ samar/parser.py`) used the wrong one; tests used both.

**FIX**: `parser.py`, `metadata_extractor.py`, and `input_representation.py`
are now thin re-exports of `core.py`. There is one canonical implementation.

### #2 Description-tokens silently mapped to `<unk>`

**WHAT**: `SamarVocab` excluded `Description_*` tokens (and used the wrong
`Average*` keys instead of FIGARO's `Mean*` keys). Every description in
training data mapped to ID 1.

**WHERE**: `samar/vocab.py` and `samar/core.py:_build_description_tokens`.

**FIX**: Matched FIGARO's design — separate `SamarVocab` (events) and
`DescriptionVocab` (description tokens). Added `DescriptionTokenizer` to
`samar/tokenizer.py`. Description tokens now use `MEAN_PITCH_KEY`/
`MEAN_VELOCITY_KEY`/`MEAN_DURATION_KEY`/`NOTE_DENSITY_KEY` exactly as in
`figaro/src/vocab.py`. `extract_metadata` still extracts
`Description_Composer_*` for human inspection but they're filtered out of
the description token stream (FIGARO never encodes free-text metadata).

### #3 `tgt=latent` instead of `latent=latent`

**WHAT**: `train_samar_transformer.SamarTransformerTrainer.forward` passed
the latent vector as the model's `tgt` parameter, bypassing
`self.latent_embedding`.

**FIX**: One-line fix at `samar/train_samar_transformer.py:45`.

### #4 `load_trained_transformer` used bare `load_state_dict`

**WHAT**: Would fail with the same state-dict mismatch we already fixed in
`generating.py`.

**FIX**: Use `SamarTransformer.from_pretrained` (same warm-start path).

## FIGARO alignment

For each design decision, FIGARO's choice was preferred where it applied:

| Decision | FIGARO | SAMAR (after fix) | Rationale |
|----------|--------|-------------------|-----------|
| `MAX_BAR_LENGTH` | 3 | 3 | matches |
| `MAX_N_BARS` | 512 | 512 | matches |
| `DEFAULT_RESOLUTION` | 480 | 480 | matches |
| `DEFAULT_POS_PER_QUARTER` | 12 | 12 | matches |
| Special tokens | `***` | `***` | matches |
| Description vocab | separate from event vocab | separate | matches |
| `token_embedding` shape | vocab_size × d_model | vocab_size × d_model | matches |
| `description_embedding` shape | separate matrix | separate matrix | matches |
| Velocity source | per-note MIDI velocity | hardcoded 64 | acceptable — SAMAR's MusicXML parser doesn't have per-note velocity; matching FIGARO would require switching from MusicXML to MIDI input |
| Token key names | `'Mean Pitch'` (space) | `'MeanPitch'` (no space) | constrained by existing pickle; will align when retrained |
| `Description_*` tokens | not produced | not produced | matches |
| PITCH_KEY | `'Pitch'` | `'Pitch_24EDO'` | intentional divergence (project's core purpose) |

## What's not yet FIGARO-aligned (deferred)

These would require either retraining or a non-trivial refactor:

1. **Token key names with spaces** — would invalidate `samar_vocab.pkl`
   and the trained checkpoint. Defer until retraining.
2. **Per-note velocity from MusicXML** — would require switching from
   MusicXML to MIDI input (matching FIGARO's pretty_midi-based pipeline).
   Not a near-term change.
3. **Description-conditional generation** — `sample()` accepts description
   now but the model was trained without it. Retraining required.
